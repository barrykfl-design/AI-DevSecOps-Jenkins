Jenkins Auto-Remediation Guide (Snyk + ZAP + Deploy)

✓ CAN Be Auto-Fixed in Jenkins
  • npm audit fix — auto-patches CVEs in Node packages
  • wp plugin update --all — updates all WordPress plugins
  • composer update — updates PHP dependencies
  • Automatic build retry on transient failures
  • Slack / email alert on failure
  • Auto-create GitHub Issue on build failure
  • Archive Snyk + ZAP reports as build artifacts
⚠ Requires Manual Review First
  • Breaking dependency upgrades (major version bumps)
  • Rotating exposed API keys and secrets
  • Fixing SQL injection / XSS in custom PHP code
  • Resolving ZAP HIGH / CRITICAL alerts in app code
  • Webhook firewall / NAT / ngrok setup
  • WordPress theme compatibility testing
  • Database schema migrations
📄  Pipeline Template — edit the 3 ENVIRONMENT lines at top, then paste into Jenkinsfile
pipeline {
    agent any
    triggers { githubPush() }
    options {
        timestamps()
        timeout(time: 30, unit: 'MINUTES')
        buildDiscarder(logRotator(numToKeepStr: '10'))
    }

    environment {
        WP_URL    = 'https://your-wordpress-site.com' // CHANGE THIS
        GH_REPO   = 'YOUR_ORG/YOUR_REPO'             // CHANGE THIS
        DEPLOY_TO = 'user@yourserver:/var/www/html/'  // CHANGE THIS
    }

    stages {

        stage('1 - Checkout') {
            steps { checkout scm }
        }

        stage('2 - Tool Check') {
            steps {
                sh '''
                    node  --version  2>/dev/null || echo "Node  NOT FOUND"
                    php   --version  2>/dev/null || echo "PHP   NOT FOUND"
                    wp    --info     2>/dev/null || echo "WP-CLI NOT FOUND"
                    snyk  --version  2>/dev/null || echo "Snyk  NOT FOUND"
                '''
            }
        }

        stage('3 - Snyk Scan') {
            steps {
                withCredentials([string(credentialsId: 'snyk-token', variable: 'SNYK_TOKEN')]) {
                    sh '''
                        snyk auth $SNYK_TOKEN
                        snyk test --severity-threshold=high || true
                        snyk monitor || true
                    '''
                }
            }
        }

        stage('4 - OWASP ZAP') {
            steps {
                sh '''
                    docker run --rm \
                        -v "$(pwd)":/zap/wrk/:rw \
                        ghcr.io/zaproxy/zaproxy:stable \
                        zap-baseline.py \
                            -t $WP_URL \
                            -r zap_report.html \
                            -x zap_report.xml \
                            --auto || true
                '''
            }
            post {
                always {
                    publishHTML([
                        allowMissing: true,
                        reportDir: '.',
                        reportFiles: 'zap_report.html',
                        reportName: 'OWASP ZAP'
                    ])
                }
            }
        }

        stage('5 - Auto-Fix CVEs') {
            parallel {
                stage('npm audit') {
                    when { expression { fileExists('package.json') } }
                    steps { sh 'npm audit fix || true' }
                }
                stage('wp plugins') {
                    when { expression { fileExists('wp-config.php') } }
                    steps { sh 'wp plugin update --all --allow-root || true' }
                }
            }
        }

        stage('6 - PHP Syntax') {
            steps {
                sh 'find . -name "*.php" -not -path "*/vendor/*" -exec php -l {} \\; 2>&1 | grep -v "No syntax errors" || true'
            }
        }

        stage('7 - Deploy') {
            when { branch 'main' }
            steps {
                withCredentials([sshUserPrivateKey(
                    credentialsId: 'deploy-key',
                    keyFileVariable: 'SSH_KEY'
                )]) {
                    sh '''
                        rsync -avz --delete \
                            --exclude=".git" \
                            --exclude="node_modules" \
                            -e "ssh -i $SSH_KEY -o StrictHostKeyChecking=no" \
                            ./ $DEPLOY_TO
                    '''
                }
            }
        }

        stage('8 - Quality Gate') {
            steps {
                script {
                    def n = sh(script: 'grep -c "High" zap_report.xml 2>/dev/null || echo 0',
                               returnStdout: true).trim().toInteger()
                    if (n > 0) { error "BLOCKED: ${n} ZAP High alerts." }
                }
            }
        }
    }

    post {
        failure {
            withCredentials([string(credentialsId: 'github-token', variable: 'GH_TOKEN')]) {
                sh 'curl -sf -X POST https://api.github.com/repos/$GH_REPO/issues -H "Authorization: token $GH_TOKEN" -d "{\\"title\\":\\"CI Fail: $JOB_NAME\\",\\"body\\":\\"$BUILD_URL\\",\\"labels\\":[\\"ci-failure\\"]}" || true'
            }
            mail(to: 'devops@yourcompany.com',
                 subject: "FAILED: ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                 body: "Log: ${env.BUILD_URL}")
        }
        always {
            archiveArtifacts(artifacts: '**/zap_report.*,**/snyk-report.json', allowEmptyArchive: true)
        }
    }
}

How to Use the Jenkinsfile Template â€” Step-by-Step Guide

❓
What exactly IS a Jenkinsfile?
A Jenkinsfile is a plain text file called Jenkinsfile (no extension) that you place in the root folder of your project on your computer (same folder as wp-config.php or package.json). It tells Jenkins what to do every time you push code to GitHub.

Think of it like a recipe â€” Jenkins reads the recipe and follows every step automatically.
❓
Can I have more than one Jenkinsfile?
Each Jenkins job can only use ONE Jenkinsfile â€” but you can have a different Jenkinsfile for each project/job.

Example setup:
  • my-wordpress-site/Jenkinsfile → used by the wordpress-deploy job
  • my-theme/Jenkinsfile → used by the theme-build job
  • security-scan/Jenkinsfile → used by the snyk-scan job
So yes â€” one per project folder = perfectly fine ✓
⚠
Will copy-pasting overwrite my original settings?
Only if you replace the entire file. You have two safe options:

Option A Merge individual stages â€” copy just the stage block you need (e.g. the Snyk stage) and paste it inside your existing pipeline's stages { } block. Nothing else changes.

Option B Replace the whole file â€” safe to do if you don't have a Jenkinsfile yet, or if you back up the old one first (rename it Jenkinsfile.bak).
📄 Step-by-Step: How to Add or Update Your Jenkinsfile
1
Find (or create) your Jenkinsfile
Open your project folder on your computer (the one connected to GitHub). Look for a file named exactly Jenkinsfile in the root.
• If it exists → open it in Notepad++ or VS Code. Do NOT delete it yet.
• If it doesn’t exist → right-click in the folder → New → Text File → rename it Jenkinsfile (remove the .txt extension).
2
Back up your existing Jenkinsfile (if you have one)
In your project folder, make a copy and name it Jenkinsfile.bak. This is your safety net â€” if anything breaks you can just rename it back.
copy Jenkinsfile Jenkinsfile.bak
3
Decide: merge a stage OR replace the whole file
✓ SAFER â€” Add one stage at a time
Find the stages { } block in your existing Jenkinsfile. Paste the new stage block (e.g. Snyk Scan, ZAP Scan) inside that block, after your last existing stage. Your other stages are untouched. 
stages {
  stage('Checkout') { ... }  ← your original
  stage('Build')    { ... }  ← your original

  // ↓ PASTE NEW STAGE HERE ↓
  stage('Snyk Scan') {
    steps {
      sh 'snyk test'
    }
  }
}
⚠ FULL REPLACE â€” Use the full template
Copy the entire template from the Auto-Remediation section above. Select ALL content in your Jenkinsfile (Ctrl+A) and paste. Before doing this: check the template has the right credentials IDs for your Jenkins setup (snyk-token, deploy-key, github-token). Change any placeholder values like YOUR_ORG and your-app-url before saving.
4
Edit the placeholder values in the template
Search the pasted code for these placeholders and replace them with your real values:
PlaceholderReplace withWhere to find it
YOUR_ORG/YOUR_REPOYour GitHub username/repo nameGitHub URL bar
your-app-urlYour WordPress site URLYour browser address bar
snyk-tokenThe credential ID you set in JenkinsManage Jenkins → Credentials
deploy-keyYour SSH deploy credential IDManage Jenkins → Credentials

5
Save the file and push to GitHub
Save the Jenkinsfile in VS Code / Notepad++. Then in GitHub Desktop:
  1. You will see Jenkinsfile listed as a changed file on the left panel
  2. Write a commit message: "ci: update Jenkinsfile with Snyk + ZAP stages"
  3. Click Commit to main
  4. Click Push origin
GitHub will send a webhook to Jenkins, which will automatically pick up the new Jenkinsfile and run it on the next build.
6
Watch the build in Jenkins
Open Jenkins in your browser (usually http://localhost:8080).
Click your job name → click the latest build number → click Console Output.
You will see each stage run in real time. If a stage turns red, the error message will appear right there â€” scroll up from the bottom to find ERROR or FAILED.
7
Something broke? Restore your backup
In your project folder, rename Jenkinsfile.bak back to Jenkinsfile (delete or rename the broken one first). Commit and push â€” Jenkins will go back to your working configuration immediately.
del Jenkinsfile && rename Jenkinsfile.bak Jenkinsfile
🔐 Quick Reference: Where to Add Credentials in Jenkins
1 Go to Jenkins → Manage Jenkins → Credentials → (global) → Add Credentials
2 For a Snyk token: Kind = Secret text → paste your Snyk API token → ID = snyk-token → Save
3 For a GitHub token: Kind = Secret text → paste GitHub Personal Access Token → ID = github-token → Save
4 For an SSH deploy key: Kind = SSH Username with private key → paste your private key → ID = deploy-key → Save
5 These IDs (snyk-token, github-token, deploy-key) must match exactly what you put in the Jenkinsfile withCredentials([...]) block
